## Architecture

This project uses a multi-stage Docker build approach:

1. **Builder Stage**: Compiles C extensions and installs dependencies
2. **Runtime Stage**: Minimal runtime environment with built extensions

The GitHub Actions pipeline includes:

- **Pre-commit Checks**: Code formatting and linting
- **Testing**: Unit, integration, and performance tests
- **Security Analysis**: Vulnerability scanning with Trivy and CodeQL
- **Documentation**: Automated API documentation generation
- **Publishing**: Docker image publishing to GitHub Container Registry
